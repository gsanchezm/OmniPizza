What's Inside

Test Users (Chaos Behaviors)

Designed with deterministic user personas for solid automated testing.

  • standard_user: Normal flow (pw: pizza123)
  • locked_out_user: Login fails (deterministic)
  • problem_user: $0 prices + broken images
  • performance_glitch_user: API delay (~3s)
  • error_user: Random checkout error (~50%)

Multi-Market Support

Simulates complex regional business rules and i18n logic.

  • MX (MXN): Requires `colonia`, Spanish (es)
  • US (USD): Requires `zip_code`, English (en)
  • CH (CHF): Requires `plz`, DE/FR Toggle
  • JP (JPY): Requires `prefectura`, Japanese (ja)

API Testing Features

Extensive header-based variations testing.

  • Headers `X-Country-Code` & `X-Language` define runtime API behaviors.
  • Fully authenticated endpoints using JWT.
  • Swagger, Redoc, and OpenAPI JSON exposed standard.

Run Locally

Backend Environment

cd backend
python -m venv venv
source venv/bin/activate
pip install -r requirements.txt
python main.py

Web Application

cd frontend
npm install
VITE_API_URL=http://localhost:8000 npm run dev

Mobile Application (Expo / RN)

cd frontend-mobile
npm install
npm run ios   # or npm run android

API Tests (Vitest)

cd tests
pnpm install
pnpm test        # Run all tests
pnpm test:ui     # Interactive UI runner

Automation Notes

  • /api/pizzas requires X-Country-Code header for market pricing.
  • Use the test users to validate reliability and chaos behaviors.
  • All interactive elements have data-testid attributes for stable selectors.
  • Checkout form fields are dynamic per market — ensure automation handles conditional rendering.
  • Phone input uses type="tel" with pattern validation (7-20 digits/spaces/+/-).
